# Инструкции по работе с планом

## Структура папки .ai-notes

```
.ai-notes/
├── INSTRUCTIONS.md                          # Инструкции: как работать с планом
├── PLAN.md                                  # Мастер-план: обзор, архитектура, фазы
├── PROGRESS.md                              # Трекер прогресса по всем фазам
├── TESTING_STRATEGY.md                      # Стратегия тестирования
└── phases/                                  # Детальные планы по каждой фазе
    ├── 01-plugin-core.md
    ├── 02-plugin-lifecycle-hooks.md
    ├── 03-plugin-config-integration.md
    ├── 04-plugin-routes-middleware.md
    ├── 05-plugin-models-data.md
    ├── 06-plugin-ui.md
    ├── 07-plugin-frontend.md
    ├── 08-plugin-controls.md
    ├── 09-plugin-discovery-loading.md
    ├── 10-fixture-plugin.md
    └── 11-testing.md
```

## Как работать с планом

### 1. Начало работы над задачей
1. Открыть PROGRESS.md → найти задачу со статусом [ ]
2. Изменить статус на [~] (в процессе)
3. Прочитать соответствующий раздел в PLAN.md и файл фазы в phases/

### 2. Во время работы
- Обновлять PROGRESS.md по мере завершения подзадач
- При блокерах — документировать в PROGRESS.md

### 3. Завершение задачи
1. Убедиться что все тесты проходят
2. Изменить статус в PROGRESS.md на [x]
3. Добавить дату завершения

## Статусы задач

| Символ | Значение        |
|--------|-----------------|
| [ ]    | Не начато       |
| [~]    | В процессе      |
| [x]    | Завершено       |
| [!]    | Заблокировано   |
| [-]    | Отменено        |

## Приоритеты

| Метка | Значение                         |
|-------|----------------------------------|
| P0    | Критично — блокирует другие      |
| P1    | Высокий — основной функционал    |
| P2    | Средний — важные улучшения       |
| P3    | Низкий — nice to have            |

## Зависимости
Не начинайте задачу, пока её зависимости не завершены.

## Ревью и тестирование
Каждая фаза должна включать:
- [ ] Unit тесты
- [ ] Интеграционные тесты (где применимо)
- [ ] Ручное тестирование
