# Field Selection (Выбор полей)

## Обзор

Функционал выбора полей позволяет указывать, какие поля должны быть загружены из базы данных при выполнении запросов. Это аналогично `attributes` в Sequelize или `select` в SQL запросах.

## Использование в фильтрах

При создании фильтра можно указать `selectedFields`:

```typescript
const filter = await FilterBuilder.create(adminizer)
    .forModel('UserAP')
    .named('Active Users')
    .selectFields(['id', 'name', 'email', 'isActive'])
    .where('isActive', 'eq', true)
    .save(user);
```

## UI в редакторе фильтра

В `FilterDialog` доступна настройка **Fields to load**.  
Если список пуст — будут загружаться все поля модели.  
Выбранные поля сохраняются в фильтре и автоматически применяются при выполнении запросов.

Подсказки:
- Используйте поиск по полям.
- Для сброса выборки очистите список.

## API

### FilterBuilder.selectFields(fields: string[])
Устанавливает поля, которые будут выбраны при выполнении фильтра.

```typescript
.selectFields(['id', 'name', 'email'])
```

### QueryParams.fields
Параметр для прямого указания полей в запросе:

```typescript
const params: QueryParams = {
    fields: ['id', 'name', 'email'],
    // ... другие параметры
};
```

## Совместимость с ORM

- **Waterline**: Использует метод `select()`
- **Sequelize**: Использует опцию `attributes`

## Преимущества

1. **Снижение объема данных**: Загружаются только необходимые поля
2. **Улучшение производительности**: Меньше данных передается по сети
3. **Оптимизация запросов**: База данных возвращает только требуемые колонки

## Обратная совместимость

Если `selectedFields` не указаны, загружаются все поля модели (поведение по умолчанию).
